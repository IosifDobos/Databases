Keys ( Primary, Foreign, Alternate )

Regarding to database design each entity must a must a have a PRIMARY KEY which is a special relation database designated to uniquey identify (unique value) and cannot contaion null values. In database each table are identified or non-udentifed from other table due to several relationship between the respective table like MANY : MANY , ONE : MANY(identified), ONE : MANY (non-undetifieed). The FOREIGN KEY basically is the PRIMARY KEY in other entity due to relantionship between repsective entities which are linked togheter trought key groups. An ALTERNATE KEY contains one or more columns whose combined values uniquely indtify every row in a table, but which is not the primary key. Each alternate key can generate a unique index or a unique constraint in database.
